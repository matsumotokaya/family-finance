# 家計簿データ分析 - プロジェクトガイド

## 📋 プロジェクト概要

このプロジェクトは、松本家のクレジットカード利用明細データを分析し、月次支出パターンやカテゴリ別の支出傾向を把握するためのものです。

## 📊 データ概要

### データの種類
- **クレジットカード利用明細書**（CSV形式）
- **対象期間**: 2025年3月〜2025年7月
- **ファイル保存場所**: `datalake/` ディレクトリ

### カードと利用者情報

#### 利用者とカード
1. **松本 夏弥さん**
   - ルミネカード（****-****-****-0021）
   - ビューカード スタンダードＭ（****-****-****-1205）

2. **松本 未来さん**
   - ビューカード スタンダードＭ（****-****-****-0406）

### ファイル命名規則
- `ご利用明細_YYYYMM.csv` - 基本形式
- `ご利用明細_YYYYMM (1).csv` - 重複ファイル（同月に複数カードがある場合など）

### 支払金額規模（参考）
- 月間支払い総額：約145,000円〜300,000円の範囲
- 利用先：日常生活費、交通費、オンライン購入、サブスクリプションなど

## 🔧 データ構造と読み取り方法

### CSVファイル構造

#### ヘッダー部分（1-5行目）
```
1行目: 会員番号,****-****-****-XXXX
2行目: 対象カード,カード名（例：ルミネカード、ビューカード スタンダードＭ）
3行目: お支払日,YYYY年MM月DD日
4行目: 今回お支払金額,"XXX,XXX"
5行目: （空行）
```

#### データ部分（6行目以降）
```
6行目: ご利用年月日,ご利用箇所,ご利用額,払戻額,ご請求額（うち手数料・利息）,支払区分（回数）,今回回数,今回ご請求額・弁済金（うち手数料・利息）,現地通貨額,通貨略称,換算レート
7行目: ****-****-****-XXXX 利用者名
8行目以降: 実際の利用データ（YYYY/MM/DD,店舗名,金額,...）
```

### 重要な注意点

#### 文字エンコーディング
- **必須**: `Shift_JIS` エンコーディングで読み込む
- UTF-8では文字化けする

#### データ読み取りのポイント
1. **利用者の特定**: 7行目以降の`****-****-****-XXXX 名前`パターンで利用者を識別
2. **金額フォーマット**: `"123,456"`形式（クォートとカンマ区切り）
3. **日付フォーマット**: `YYYY/MM/DD`形式
4. **複数利用者**: 1つのファイル内に複数の利用者データが含まれる場合がある

## 🏷️ 利用先カテゴリ分類

### 判明している主要カテゴリ
1. **🚗 交通費（ETC）**: 各種高速道路利用料金
2. **🛒 オンライン購入**: Amazon、Apple、AI関連サービス
3. **🍽️ 飲食**: レストラン、ファストフード
4. **🏪 小売・日用品**: ニトリ、オートバックス、コンビニ
5. **💼 保険・金融**: 生命保険料、カード年会費
6. **📱 通信費**: ソフトバンク
7. **💊 薬局・ドラッグストア**: クリエイト等
8. **👶 子供用品・衣料**: 子供服等
9. **🎵 エンターテイメント**: 音楽配信等

## ⚠️ 既知の問題と注意点

### データ処理時の注意
1. **金額データの正確性**: 支払金額は十万円台が正常（数百円は異常）
2. **利用者の特定**: 同一ファイル内に複数利用者のデータが混在
3. **カテゴリ分類**: 店舗名から自動分類は限界があるため手動調整が必要
4. **重複ファイル**: 同月に複数ファイルがある場合は内容を確認して統合

### 分析の観点
- **利用者別**: 夏弥さんは日常生活費中心、未来さんはオンライン購入中心
- **月次トレンド**: 季節性や特別な支出の把握
- **カテゴリ別**: 固定費vs変動費の分析

## 🚀 分析開始手順

1. **文字コード確認**: Shift_JISで読み取り可能か確認
2. **データ構造確認**: ヘッダー情報と利用者情報を正しく抽出
3. **金額データ検証**: 支払金額が適切な範囲（十万円台）にあるか確認
4. **カテゴリ分類**: 利用先を適切なカテゴリに分類
5. **月次・利用者別分析**: トレンドと傾向の把握
